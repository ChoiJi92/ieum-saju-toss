# 이음사주 v2 첫날 몰입·재방문·공유 개선 설계

## 목표

정령을 깨운 사용자가 첫 운세에서 끝나지 않고 첫 교감과 성장까지 경험하게 한다. 이후에는 다음 출석 보상을 명확히 예고하고, 정령이 포함된 오늘의 운세 카드를 부담 없이 공유할 수 있게 한다.

## 유지 원칙

- 오늘의 운세와 정령 깨우기·교감·성장은 무료로 유지한다.
- 기존 상세 운세 화면의 보상형 광고 정책은 변경하지 않는다.
- 광고는 정령 성장의 필수 조건이 아니며 기존 선택형 성장 보너스를 유지한다.
- 기존 디자인 시스템과 정령 탄생 애니메이션을 유지한다.
- 새로운 서버나 결제 시스템을 추가하지 않는다.

## 접근 방식

### 채택: 기존 상태를 연결하는 단계형 개선

이미 구현된 `SpiritProgress`, 일일 미션, 출석 스트릭, 정령 카드 생성기를 활용한다. 새로운 포인트나 별도 퀘스트 저장소를 만들지 않고 화면의 안내, 행동 순서, 완료 피드백을 개선한다. 변경 범위가 작고 기존 사용자 데이터와 호환된다.

### 제외한 대안

- 별도 첫날 퀘스트 데이터 모델: 구현과 마이그레이션 비용에 비해 현재 3개 행동으로 충분하다.
- 연속 출석 전용 캘린더: 홈의 핵심인 정령과 운세보다 화면을 무겁게 만든다.
- 공유 보상 지급: 자발적인 공유보다 보상 수집 행동을 유도하고 중복 지급 방지가 필요하다.

## 사용자 흐름

### 1. 첫날 몰입

1. 세 번 교감하여 정령을 깨운다.
2. 정령 소개 CTA로 무료 오늘의 운세를 연다.
3. 운세 하단에서 `정령에게 오늘 기운 전하기` CTA를 누르면 홈으로 돌아간다.
4. 홈의 첫날 가이드는 아직 하지 않은 무료 교감 하나를 강조한다.
5. 교감 완료 후 성장 게이지 변화와 다음 성장까지 남은 양을 보여준다.

오늘 운세 확인은 기존 `fortune` 보너스로 기록한다. 첫날 안내는 기존 진행 상태에서 파생하며 별도 완료 플래그를 만들지 않는다.

### 2. 재방문

홈 상단에 현재 연속 방문 일수와 다음 마일스톤을 함께 표시한다. 예: `2일 연속 · 내일 +20`. 당일 첫 진입 보상과 3·7·14·30일 마일스톤 보상은 기존 로직을 그대로 사용한다.

일일 미션 카드에는 미완료 항목별 다음 행동을 제공한다. 오늘 운세는 운세 화면, 정령 잡기는 도감, 교감은 해당 버튼 영역으로 이동한다. 완료 후 기존 `+15 교감` 보상을 받는다.

### 3. 공유

오늘의 운세 화면 하단에 `오늘 운세 카드 공유`를 배치한다. 공유 카드에는 정령 이미지, 정령 이름, 오늘의 한마디, 이음사주 브랜드만 포함하고 생년월일 등 개인정보는 넣지 않는다.

홈 헤더의 정령 카드 공유는 유지한다. 두 진입점은 같은 카드 생성기를 사용하되 문구만 화면 맥락에 맞춘다. 공유 취소는 오류 알림을 띄우지 않고, 실제 생성 실패만 재시도 안내를 표시한다.

## 상태와 오류 처리

- 모든 첫날·미션 상태는 `SpiritProgress`에서 파생한다.
- 스트릭은 기존 `ieum-saju.streak.v1`을 사용한다.
- 공유 이미지를 만들지 못하면 현재 화면에 짧은 실패 안내를 표시한다.
- 로컬 저장소 접근 실패 시 핵심 운세 열람과 화면 이동은 계속 가능해야 한다.
- 기존 사용자와 부화 완료 사용자의 진행 상태를 초기화하지 않는다.

## 검증

- 신규 사용자: 부화 → 오늘 운세 → 홈 → 첫 교감까지 모바일 브라우저에서 확인한다.
- 기존 사용자: 홈 진입 시 스트릭과 다음 보상이 올바르게 보이는지 확인한다.
- 미션 각 항목의 이동 대상과 완료 상태를 확인한다.
- 공유 카드에 정령과 오늘의 한마디가 포함되고 개인정보가 없는지 확인한다.
- 린트와 프로덕션 AIT 빌드를 실행한다.

## 후속 범위

실제 전환율 측정은 서버 이벤트 수신 API가 준비된 뒤 `spirit_hatched`, `today_viewed`, `first_care_completed`, `share_completed` 이벤트로 추가한다. 수신점 없이 로컬 로그만 쌓는 구현은 하지 않는다.
